よくある質問（FAQ）

FLMに関するよくある質問と回答をまとめました。お探しの情報が見つからない場合は、お問い合わせからご連絡ください。

📋 基本情報

FLMとは何ですか？

FLMは、誰でも簡単にローカルLLMのAPIを作成・管理できるデスクトップアプリケーションです。 プログラミングの知識がなくても、GUIで操作するだけで、Ollamaを使ったAI APIを数分で作成できます。

すべてのデータ処理はローカルで行われ、インターネットにデータを送信することはありません。 そのため、プライバシーを完全に保護しながら、AIを活用できます。

誰が使えますか？（非開発者でも使えますか？）

はい、非開発者でも使えます。FLMは、技術的な知識がなくても使えるように設計されています。

  • プログラミングの知識は不要です
  • コマンドライン操作は不要です
  • すべての操作がGUIで完結します
  • 直感的なインターフェースで、すぐに使い始められます
料金はかかりますか？

FLMは完全に無料で利用できます。将来的にも無料で提供される予定です。

追加のサービスやプレミアム機能を検討する予定はありません。 すべての機能が無料で利用できます。

💻 インストール・セットアップ

どのOSで動作しますか？

FLMは、以下のOSで動作します：

  • Windows: Windows 10/11 (64-bit)
  • macOS: macOS 11.0+ (Intel & Apple Silicon対応)
  • Linux: Ubuntu 20.04+、Debian 11+、Fedora 34+
システム要件は？

最小要件:

  • OS: Windows 10/11、macOS 11.0+、Ubuntu 20.04+
  • RAM: 8GB
  • ストレージ: 10GB（モデル含む）
  • CPU: 64-bit プロセッサ

推奨要件:

  • RAM: 16GB以上
  • ストレージ: 20GB以上
  • GPU: NVIDIA GPU（オプション、高速化のため）
インストール方法は？

ダウンロードページから、お使いのOS向けのインストーラーをダウンロードしてください。

  • Windows: .exeファイルを実行し、インストールウィザードに従います
  • macOS: .dmgファイルを開き、アプリケーションフォルダにドラッグ&ドロップします
  • Linux: .AppImageファイルに実行権限を付与して実行します

詳細な手順は、使い方ガイドを参照してください。

📖 使い方

APIの作成方法は？

APIの作成は非常に簡単です。以下の3ステップで完了します：

  1. ホーム画面から「新しいAPIを作成」ボタンをクリック
  2. モデル選択画面で、使用したいAIモデルを選択
  3. 設定画面で、API名やポート番号を設定（必要に応じて変更）し、「作成」ボタンをクリック

詳細な手順は、使い方ガイドの「ステップ4: API作成」を参照してください。

モデルはどこからダウンロードしますか？

モデルは、FLMアプリ内の「モデル管理」画面からダウンロードできます。

  • ホーム画面から「モデル管理」を選択
  • 「モデル検索・ダウンロード」タブで、必要なモデルを検索
  • モデルを選択して「ダウンロード」ボタンをクリック

モデルはOllama公式ライブラリから自動でダウンロードされます。 ダウンロードには数分から数十分かかることがあります（モデルのサイズによります）。

APIを他のアプリで使う方法は？

APIを作成した後、詳細画面に表示されるエンドポイントURLとAPIキーを使って、他のアプリケーションからAPIを呼び出せます。

詳細画面には、以下のサンプルコードが表示されます：

  • curlコマンド（コマンドライン用）
  • Pythonコード
  • JavaScriptコード

これらのサンプルコードをコピーして、お使いのアプリケーションに貼り付けて使用できます。

APIとは何ですか？

API（Application Programming Interface）は、アプリケーション間でデータをやり取りするためのインターフェースです。 FLMで作成したAPIを使用して、他のアプリケーションからLLMの機能を利用できます。

例えば、チャットボットや自動応答システム、コード生成支援ツールなど、さまざまなアプリケーションからFLMで作成したAPIを呼び出して使用できます。

どのモデルを選べばいいですか？

用途に応じて最適なモデルを選ぶことができます：

  • 初心者: 推奨モデルを選択（画面上で⭐マークで表示）
  • チャット用途: llama3:8b、mistral:7bなど
  • コード生成: codellama:7b、deepseek-coder:6.7bなど
  • 軽量モデル: tinyllama:1.1b、phi:2.7bなど（メモリが少ない環境向け）

モデル管理画面では、モデルのサイズ、パラメータ数、カテゴリなどの情報を確認できます。

ポート番号は何を選べばいいですか？

デフォルトの8080で問題ありません。他のアプリケーションが使用している場合は、8081、8082などの別のポート番号を選択してください。

ポート番号は1024から65535の範囲で選択できます。よく使用されるポート番号（80、443、3306など）は避けることをおすすめします。

WEBサイトに設定できる使い捨てのAPIを作成できますか？

はい、HTML/CSS/JSだけで動作する静的WEBサイトに設定できる使い捨てのAPIを簡単に作成できます。

使い捨てAPIとは:

  • 一時的なプロジェクトやデモサイト、プロトタイプ開発などで短期間だけ必要なAPI
  • 数分で作成・削除可能
  • 設定が簡単で、すぐに利用開始できる
  • 不要になったら簡単に削除できる

活用例:

  • デモサイト: クライアントへのデモ用に一時的なAPIを作成。デモ終了後はすぐに削除
  • プロトタイプ開発: アイデアの検証用に素早くAPIを作成。検証後は不要になったら削除
  • 学習・教育: 学習用の一時的なAPIを作成。学習終了後は簡単に削除

詳細な手順は、使い方ガイドの「ステップ6: WEBサイト用使い捨てAPIの作成」を参照してください。

このWEBサイトは静的サイトですか？

はい、このWEBサイト自体がHTML/CSS/JSだけで動作する静的サイトです。

静的サイトの特徴:

  • サーバーサイド処理不要: HTML、CSS、JavaScriptのみで動作
  • どこでもホスティング可能: GitHub Pages、Netlify、Vercelなど、あらゆる静的ホスティングサービスで利用可能
  • 高速で安全: 静的ファイルのみのため、高速で安全に配信
  • メンテナンスが簡単: ファイルを編集するだけで更新可能

このWEBサイトのすべてのページがHTMLファイルとして存在し、CSSとJavaScriptは外部ファイルとして読み込まれます。 サーバーサイドの処理やデータベースは一切使用していません。

詳細は、使い方ガイドの「ステップ7: このWEBサイトについて（静的サイト）」を参照してください。

認証を有効にする必要はありますか？

認証の有効化は、使用環境によって異なります：

  • ローカル環境のみで使用: 認証を無効にしても問題ありません
  • 外部アプリケーションから使用: 認証を有効にすることをおすすめします（セキュリティのため）

認証を有効にすると、APIキーが必要になります。APIキーは作成時に表示されるので、安全な場所にコピーして保存してください。

APIキーはどこで確認できますか？

APIキーは以下の方法で確認できます：

  • API作成時: 成功画面に表示されます
  • API詳細画面: 「詳細」ボタンから確認できます（表示/非表示を切り替え可能）
  • 設定変更画面: APIキーを再生成できます

重要: APIキーは作成時に一度だけ表示されます。安全な場所にコピーして保存してください。

Ollamaとは何ですか？

Ollamaは、ローカルでLLM（大規模言語モデル）を実行するためのツールです。FLMはOllamaを使用してLLMのAPIを提供します。

Ollamaは、あなたのコンピューター上でAIモデルを実行するため、インターネットにデータを送信することなく、プライバシーを完全に保護しながらAIを活用できます。

Ollamaのインストールは必要ですか？

いいえ、必要ありません。FLMが自動的にOllamaをダウンロード・インストールします。既にインストールされている場合は、それを使用します。

初回起動時に、Ollamaのセットアップが自動で開始されます。インターネット接続が必要です。

モデルのサイズはどのくらいですか？

モデルによって異なります：

  • 小: 1-3GB（例: tinyllama）
  • 中: 4-7GB（例: llama3:8b）
  • 大: 8GB以上（例: llama3:70b）

モデル管理画面では、各モデルのサイズが表示されます。ストレージ容量を確認してからダウンロードしてください。

モデルのダウンロードに時間がかかりますか？

はい、モデルのサイズとインターネット速度によって異なります：

  • 小: 数分
  • 中: 10-30分
  • 大: 1時間以上

ダウンロード中は、プログレスバーで進捗状況を確認できます。ダウンロードは一時停止・再開・キャンセルが可能です。

モデルを削除できますか？

はい、インストール済みモデル一覧から削除できます。削除すると、ストレージが解放されます。

モデルを削除するには、モデル管理画面の「インストール済みモデル」タブで、削除したいモデルの「削除」ボタンをクリックします。

注意: 削除したモデルを使用しているAPIがある場合、そのAPIは正常に動作しなくなる可能性があります。

🔧 トラブルシューティング

APIが起動しません

以下の点を確認してください：

  • Ollamaが正しく起動しているか（ホーム画面で確認）
  • ポート番号が他のアプリケーションと重複していないか（別のポート番号を試してください）
  • モデルが正しくダウンロードされているか（モデル管理画面で確認）
  • インターネット接続が安定しているか（初回セットアップ時）

問題が続く場合は、お問い合わせからご連絡ください。

エラーメッセージが表示されます

よくあるエラーと解決方法：

  • 「ポート番号が既に使用されています」→ 別のポート番号を試してください
  • 「Ollamaに接続できません」→ Ollamaが起動しているか確認してください
  • 「モデルが見つかりません」→ モデルがダウンロードされているか確認してください
  • 「APIキーの再生成に失敗しました」→ 認証が有効になっているか確認してください
モデルのダウンロードに失敗します

以下の点を確認してください：

  • インターネット接続が安定しているか
  • ストレージに十分な空き容量があるか（モデルは数GBになることがあります）
  • ファイアウォールがダウンロードをブロックしていないか
  • プロキシ設定が必要な場合、システムのプロキシ設定を確認してください

問題が続く場合は、お問い合わせからご連絡ください。

APIキーを忘れてしまいました

APIキーを再生成できます。設定変更画面で「APIキーを再生成」ボタンをクリックしてください。

注意: 再生成すると、古いAPIキーは無効になります。新しいAPIキーを使用するように、接続しているアプリケーションの設定を更新してください。

エラーメッセージがわかりません

FLMのエラーメッセージは非開発者向けに作成されています。メッセージの内容に従って操作してください。

問題が解決しない場合は、以下の情報を確認してください：

  • 使い方ガイドの「トラブルシューティング」セクション
  • このFAQのトラブルシューティングセクション
  • お問い合わせからサポートに連絡

⚙️ 技術的な質問

FLMはオフラインで動作しますか？

部分的にオフラインで動作します：

  • モデルダウンロード: インターネット接続が必要です（初回のみ）
  • APIの利用: オフラインで動作します（モデルがダウンロード済みの場合）

モデルをダウンロードした後は、インターネット接続なしでもAPIを使用できます。

複数のAPIを同時に起動できますか？

はい、異なるポート番号を使用すれば、複数のAPIを同時に起動できます。

例えば、ポート8080でチャット用API、ポート8081でコード生成用APIというように、用途に応じて複数のAPIを運用できます。

APIはOpenAI互換ですか？

はい、OpenAI互換のAPIエンドポイントを提供します。OpenAIのSDKを使用してAPIに接続できます。

FLMで作成したAPIは、OpenAIの`/v1/chat/completions`エンドポイントと互換性があるため、OpenAIのSDKを使用しているアプリケーションからそのまま使用できます。

データはどこに保存されますか？

データは以下の場所に保存されます：

  • Windows: %APPDATA%\FLM\
  • macOS: ~/Library/Application Support/FLM/
  • Linux: ~/.local/share/FLM/

このディレクトリには、データベースファイル、設定ファイル、Ollamaのポータブル版（自動ダウンロードした場合）などが保存されます。

⚡ パフォーマンス

APIの応答速度はどのくらいですか？

モデルとハードウェアによって異なります：

  • 軽量モデル: 数秒
  • 中規模モデル: 10-30秒
  • 大規模モデル: 1分以上

GPU（特にNVIDIA GPU）を使用すると、応答速度が大幅に向上します。

メモリ使用量はどのくらいですか？

モデルサイズに応じて異なります：

  • 小: 2-4GB
  • 中: 8-12GB
  • 大: 16GB以上

複数のAPIを同時に起動する場合は、それぞれのメモリ使用量が加算されます。

CPU使用率が高いです

LLMの推論処理はCPUを多く使用します。これは正常な動作です。

必要に応じて、以下の対策を試してください：

  • 軽量モデルを使用する
  • GPUを使用する（NVIDIA GPU推奨）
  • 同時に実行するAPIの数を減らす

🔒 セキュリティ・プライバシー

データは外部に送信されますか？

いいえ、データは一切外部に送信されません。すべてのデータ処理はローカルで行われます。

  • AIモデルはすべてローカルで実行されます
  • APIへのリクエストもローカルで処理されます
  • インターネットへのデータ送信は発生しません
  • Ollamaのダウンロード時のみ、インターネット接続が必要です（一度だけ）

そのため、完全にプライバシーが保護されます。

APIキーは安全ですか？

はい、APIキーは安全に管理されます：

  • APIキーは強力な暗号化アルゴリズム（AES-256-GCM）で暗号化して保存されます
  • APIキーはデータベースにハッシュ値として保存されます
  • APIキーは作成時に一度だけ表示されます（安全な場所にコピーして保存してください）
  • APIキーを忘れた場合は、再生成できます（古いキーは無効になります）

重要: APIキーは他人に共有しないでください。

ローカルで実行されることは保証されますか？

はい、保証されます。FLMは以下の設計により、すべての処理がローカルで完結するようになっています：

  • Ollamaはローカルで実行されます（デフォルト: localhost:11434）
  • 認証プロキシもローカルで実行されます（デフォルト: localhost:8080など）
  • データベース（SQLite）もローカルに保存されます
  • すべての通信はローカルホスト（127.0.0.1）内で完結します

ソースコードは将来的に公開予定ですので、動作を確認できます。
