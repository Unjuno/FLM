# FLM - 問題リスト（潜在的な問題・デザイン問題）

## 📋 使用方法

この問題リストは、複数のAIエージェントが並行して問題を確認・修正を行うためのものです。

### 作業フロー

1. **自分のロールを確認**: どのエージェントロールで作業するか確認
2. **担当ロールの問題を優先**: 自分のロールに合った問題を確認
3. **ステータス確認**: `🟡 未確認` または `🔵 確認中` の問題を確認
4. **問題の確認・調査**: 実際に問題が存在するか確認
5. **ステータス更新**: 
   - 確認開始時: `🟡 未確認` → `🔵 確認中` に変更し、担当エージェント名を記載
   - 確認完了時: `🔵 確認中` → `✅ 確認済み（問題なし）` または `⚠️ 確認済み（問題あり）` に変更
   - 修正完了時: `⚠️ 確認済み（問題あり）` → `✅ 修正済み` に変更

### ステータス凡例

- 🟡 **未確認**: まだ誰も確認していない
- 🔵 **確認中**: 現在エージェントが確認中（エージェント名を記載: 例 `🔵 確認中 - QA`）
- ✅ **確認済み（問題なし）**: 確認したが問題なし
- ⚠️ **確認済み（問題あり）**: 確認した結果、問題が存在する
- ✅ **修正済み**: 問題を修正済み

### 優先度凡例

- 🔴 **高**: セキュリティ問題、データ損失リスク、重大なバグ
- 🟡 **中**: パフォーマンス問題、UX問題、設計の問題
- 🟢 **低**: コード品質、ドキュメント、軽微な改善
