### 2.1 タイトル作成

**推奨フォーマット**:
```
FLM – One-click local LLM API creation for non-developers
```

**タイトルのコツ**:
- ✅ プロダクト名を含める
- ✅ 主要な価値提案を簡潔に

- ❌ 技術用語の多用を避ける

**タイトル候補**:
1. "FLM – Create and use local LLM APIs with one click, no coding required"
2. "FLM – Making local LLM APIs accessible to everyone"
3. "Show HN: FLM – One-click local LLM API creation (no coding required)"

### 2.2 本文作成

**推奨構成**:

#### パターン1: ストーリー重視

```markdown
We built FLM around a simple principle: **anyone should be able to create and use APIs safely and easily**.

Deploying local LLMs used to require hours of manual configuration, security hardening, and deployment scripts. Now it's one click to create an API, and one click to use it.

**What it does:**
- Create local LLM APIs with one click (no coding required)

Many people want to use local LLMs for privacy and cost reasons, but the technical barrier is too high. We wanted to make this accessible to everyone, not just developers.

**Tech stack:**
- Tauri (cross-platform desktop app)
- Ollama (LLM execution engine)
- Express.js (authentication proxy)
- SQLite (local storage)

Try it out: [Website URL] | [GitHub URL]

Feedback welcome!
```

#### パターン2: 技術詳細重視

```markdown
Show HN: FLM – One-click local LLM API creation tool

FLM is a desktop application that lets you create and use local LLM APIs without writing any code.

**Key features:**
- One-click API creation (select model → click "Create" → done)
- Built-in authentication (API keys auto-generated)
- OpenAI-compatible API endpoints
- GUI for non-developers, REST API for developers

**How it works:**
1. Select a local LLM model (via Ollama)
2. Configure API name, port, authentication
3. Click "Create API"
4. Get a secure API endpoint with API key

**Technical details:**
- Frontend: Tauri + React/Vue
- Backend: Ollama (REST API)
- Auth: Express.js proxy with API key validation
- Storage: SQLite for API configurations

**Why we chose Ollama:**
We wanted to leverage existing OSS instead of reinventing the wheel. Ollama provides excellent REST API and OpenAI compatibility out of the box.

**Use cases:**
- Non-developers who want to add AI features to their apps
- Developers who want privacy-focused LLM APIs
- Anyone who wants to avoid cloud LLM API costs

**Current status:**
MVP released (Windows only). macOS/Linux support coming in v1.0.

Try it: [Website] | Code: [GitHub]
```

## 投稿手順

### 3.1 Hacker Newsアカウント作成

1. https://news.ycombinator.com/ にアクセス
2. 「login」をクリック
3. アカウント作成（必要に応じて）
4. プロフィール設定（任意、信頼性向上のため推奨）

### 3.2 投稿タイミング

**最適なタイミング**:
- 太平洋時間（PT）の平日 午前6-10時
- 日本時間の平日 午後11時-翌朝3時

**避けるべきタイミング**:
- 週末（反応が少ない）
- 深夜（PT基準）

### 3.3 投稿手順

1. https://news.ycombinator.com/submit にアクセス
2. 「title」: タイトルを入力
3. 「url」: 公式WebサイトまたはGitHubのURLを入力
4. 「text」: 本文を入力（URL入力時は「Ask HN」形式になるため注意）
5. 「submit」をクリック

**注意事項**:
- 「Show HN」形式で投稿する場合、タイトルに「Show HN:」を含める
- URLとテキストの両方を入力する場合、テキストが優先される
- スパム判定を避けるため、過度な宣伝は避ける

### 3.4 URL vs Text投稿

**URL投稿（推奨）**:
- 公式WebサイトのURLを入力
- Webサイトに詳細説明を記載
- Hacker Newsには簡潔な説明のみ

**Text投稿（Ask HN形式）**:
- URLを入力せず、テキストのみ
- 本文内にURLを記載可能
- より詳細な説明が可能

**推奨**: URL投稿 + Webサイトに詳細記載

## よくある質問への対応

### 5.1 技術的質問

**Q: Why not use [他のツール]?**
```
A: We evaluated [他のツール], but chose Ollama because:
- It provides excellent REST API out of the box
- OpenAI compatibility built-in
- Active community and maintenance
- We follow OSS-first approach
```

**Q: How is this different from Ollama?**
```
A: Ollama is the LLM execution engine. FLM is a GUI wrapper that:
- Makes Ollama accessible to non-developers
- Adds authentication layer (API keys)
- Provides simple interface (no CLI required)
- We use Ollama's API directly - we don't reinvent the wheel
```

**Q: Is it open source?**
```
A: [状況に応じて]
- Yes, MIT license: [GitHub URL]
- Not yet, but we're considering open sourcing
```

### 5.2 機能に関する質問

**Q: When will macOS/Linux support be available?**
```
A: macOS and Linux support is planned for v1.0, which should be available in 2-3 months after MVP. We're currently focusing on Windows for the MVP.
```

**Q: Can I use my own models?**
```
A: Yes! FLM works with any model that Ollama supports. You can pull custom models using Ollama's API, or we're planning to add GUI support in future versions.
```

**Q: Is it secure?**
```
A: Yes, we implement:
- API key authentication (auto-generated, 32+ characters)
- Local-only access (default: localhost)
- Encrypted API key storage
- HTTPS support planned for future versions
```
