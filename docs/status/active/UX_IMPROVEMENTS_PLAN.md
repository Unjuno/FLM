# 非開発者向けUX改善計画

> Status: Active | Updated: 2025-01-27 | Audience: UI engineers

## 概要

FLMを大衆向け（非開発者向け）にするため、専門用語の排除と直感的な操作フローの実装を行います。

## 実装状況

### ✅ 実装済み
- セキュリティUI（監査ログ、侵入検知、IPブロックリスト、異常検知）
- Chat Tester UI
- 基本的なルーティング
- **用語マッピングの追加**（`locales/ja.json`, `locales/en.json`に`terminology`セクションを追加）
- **主要ページの翻訳キー追加**（Dashboard, Chat Tester, Securityの翻訳キーを追加）
- **ヘルプテキストの追加**（各ページのヘルプセクション、よくある質問を追加）
- **エラーメッセージの改善**（技術的エラーを分かりやすい表現に変換、エラーメッセージの翻訳キーを追加、解決方法の提示を追加）
- **確認ダイアログの翻訳キー追加**（アクセスキー削除、IPブロック、サーバー停止などの確認ダイアログ）
- **状態の可視化の翻訳キー追加**（ローディング、成功、失敗、プログレスバーなどの状態表示）
- **セットアップウィザードの翻訳キー追加**（初回起動時のガイド、サーバー設定、セキュリティ設定）

### ⏳ 改善が必要
- 主要ページの用語置き換え（UIコンポーネント実装時に適用）
- 確認ダイアログと状態の可視化のUI実装（翻訳キーは準備済み）

## 改善タスク

### 1. 専門用語の置き換え

#### 用語マッピング
| 技術用語 | 非開発者向け表現 | 使用箇所 |
|---------|----------------|---------|
| エンドポイント | 接続先URL | Chat Tester, Dashboard |
| デプロイ | 公開・起動 | API Setup |
| API | サービス | 全体 |
| プロキシ | サーバー | Dashboard |
| リクエスト | 送信 | Chat Tester |
| レスポンス | 応答 | Chat Tester |
| 認証 | ログイン・アクセス許可 | Security |
| ホワイトリスト | 許可リスト | Security |
| ブロックリスト | 拒否リスト | Security |
| レート制限 | アクセス制限 | Security |
| 監査ログ | アクセス記録 | Security |
| 侵入検知 | 不正アクセス検出 | Security |
| 異常検知 | 異常な動作の検出 | Security |

#### 実装方法
1. `locales/ja.json` と `locales/en.json` に用語マッピングを追加
2. 既存のUIコンポーネントで用語を置き換え
3. ツールチップやヘルプテキストで技術用語の説明を追加

### 2. 直感的な操作フローの改善

#### 改善ポイント
1. **初回起動時のガイド**
   - セットアップウィザードの改善
   - 各ステップでの説明文の追加
   - 視覚的なガイド（アイコン、矢印など）

2. **エラーメッセージの改善**
   - 技術的なエラーメッセージを分かりやすい表現に変換
   - 解決方法の提示
   - ヘルプページへのリンク

3. **操作の確認**
   - 重要な操作（APIキー削除、IPブロックなど）での確認ダイアログ
   - 操作の結果を明確に表示

4. **状態の可視化**
   - ローディング状態の明確な表示
   - 成功/失敗の視覚的フィードバック
   - プログレスバーの追加

### 3. ヘルプテキストの追加

#### 実装箇所
1. **各ページのヘルプセクション**
   - ページ上部に「？」アイコンを配置
   - クリックでヘルプモーダルを表示

2. **フォームフィールドの説明**
   - 各入力フィールドに説明テキストを追加
   - プレースホルダーテキストの改善

3. **ツールチップ**
   - 専門用語にマウスオーバーで説明を表示
   - アイコンやボタンにツールチップを追加

### 4. エラーハンドリングの改善

#### 改善内容
1. **エラーメッセージの変換**
   - 技術的なエラーコードを分かりやすいメッセージに変換
   - ユーザーが取るべきアクションを提示

2. **エラー回復の支援**
   - 「再試行」ボタンの追加
   - エラー発生時の代替手段の提示

## 実装順序

1. **用語マッピングの追加**（`locales/ja.json`, `locales/en.json`）
2. **主要ページの用語置き換え**（Dashboard, Chat Tester, Security）
3. **ヘルプテキストの追加**（各ページ）
4. **エラーメッセージの改善**（全体）
5. **操作フローの改善**（セットアップウィザード、確認ダイアログ）
